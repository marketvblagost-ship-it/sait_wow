## Telegram-заявки

Статичный сайт на GitHub Pages не может сам отправить заявку в личный Telegram без серверного обработчика.

Сейчас сделан безопасный рабочий сценарий:

1. Посетитель проходит квиз.
2. Оставляет Telegram или телефон.
3. Сайт собирает текст заявки.
4. Текст копируется в буфер.
5. Открывается Telegram @AmalTarget.
6. Пользователь отправляет готовое сообщение.

## Для автоматической отправки прямо вам в Telegram

Нужен промежуточный обработчик:

сайт → webhook / Google Apps Script / сервер → Telegram Bot API

В `script.js` есть строка:

```js
const LEAD_ENDPOINT = '';
```

Когда будет готов webhook, вставьте URL:

```js
const LEAD_ENDPOINT = 'https://ВАШ_ОБРАБОТЧИК_ЗАЯВОК';
```

Нельзя вставлять Telegram Bot Token прямо в `script.js`, потому что его увидят посетители сайта.

## Проверка перед публикацией

1. Открыть `index.html` локально.
2. Нажать все кнопки «Получить макет».
3. Пройти квиз до конца.
4. Проверить, что Telegram открывается.
5. Проверить, что текст заявки скопирован и виден на финальном экране.
6. После публикации проверить цели в Яндекс.Метрике:
   - quiz_open
   - quiz_step_1
   - quiz_step_2
   - quiz_step_3
   - quiz_step_4
   - quiz_submit
   - get_mockup
   - click_telegram
   - click_whatsapp
